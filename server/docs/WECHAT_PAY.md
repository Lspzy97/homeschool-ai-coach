# 微信支付接入指南

## 方案A：微信商户号支付（正规方案）

### 前置要求
1. 已有微信支付商户号（商户号、API密钥、证书）
2. 服务器域名已备案并配置HTTPS

### 配置步骤

1. **上传证书文件**
   将微信支付商户平台下载的证书文件上传到服务器：
   - `apiclient_cert.pem` - 证书
   - `apiclient_key.pem` - 私钥

2. **配置环境变量**
   ```bash
   cp .env.example .env
   # 编辑 .env 填入您的信息
   ```

3. **填入配置信息**
   - `WECHAT_MCH_ID` - 商户号
   - `WECHAT_SERIAL_NO` - 证书序列号
   - `WECHAT_PRIVATE_KEY` - 私钥文件路径
   - `WECHAT_API_KEY` - API密钥
   - `WECHAT_APPID` - 小程序AppID
   - `WECHAT_NOTIFY_URL` - 支付回调地址

4. **配置支付回调**
   在微信支付商户平台设置支付回调地址：
   - 路径：`https://你的域名/api/payment/callback`
   - 需通过微信支付验收

## 方案B：个人收款码（简化方案）

此方案生成收款二维码，用户扫码转账后人工确认开通。

### 实现思路
1. 后端生成带订单号的收款二维码
2. 用户转账时备注订单号
3. 作者收到转账后后台手动开通

---

## 方案C：维持现状（人工转账）

当前已实现的方案：
1. 用户复制作者微信号 `mss871386855`
2. 添加后转账 9.9 元
3. 作者后台手动开通

此方案优点：
- 无需技术对接
- 无微信手续费
- 资金直接到账

缺点：
- 用户体验稍差
- 无法自动化

## 支付配置检测

启动服务器后，访问：
```
GET /api/payment/config
```

返回 `configured: true` 表示微信支付已配置成功。
